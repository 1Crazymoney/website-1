Think of IPFS as bittorrent + git

Here's what happens when you add a folder of files to IPFS:
The folder, its files, and all of the blocks within each file are given a unique index called a hash.
Because every block and every file in IPFS has a unique hash, we're able to remove duplications across the network and track a version history for every file.
On the network, each node is aware of each of its neighbors, and has a list of the location of the hashes they know about.
When you're looking up a file or a folder, you look it up by its hash. If a node doesn't know about a hash, it asks the network to find nodes that contain the hash and returns it to you.
To make things easy for people, everything can be found by human-readable names, using a decentralized naming system called IPNS.
