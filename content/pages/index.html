The web of tomorrow needs IPFS today

HTTP is inefficient and expensive

HTTP downloads a file from a single computer at a time, instead of getting pieces from multiple computers simultaneously. With video delivery, a P2P approach could save 60% in bandwidth costs.

Humanity's history is deleted daily

The average lifespan of a web page is 100 days. Remember GeoCities? The web doesn't anymore. It's not good enough for the primary medium of our era to be so fragile.

The web's centralization limits freedom

Facebook and Youtube account for 40% of all mobile traffic. It's too easy for powerful forces and reactive politics to undermine the web—against the common interests of humanity.

We need diversely resilient networks

Developing world. Offline. Natural disasters. Intermittent connections. All trivial compared to interplanetary networking. We need networks with persistent availability without Internet backbone connectivity.

IPFS aims to replace HTTP and
build a better web for all of humanity.

Think of IPFS as bittorrent + git

Here's what happens when you add a folder of files to IPFS:
The folder, its files, and all of the blocks within each file are given a unique index called a hash.
Because every block and every file in IPFS has a unique hash, we're able to remove duplications across the network and track a version history for every file.
On the network, each node is aware of each of its neighbors, and has a list of the location of the hashes they know about.
When you're looking up a file or a folder, you look it up by its hash. If a node doesn't know about a hash, it asks the network to find nodes that contain the hash and returns it to you.
To make things easy for people, everything can be found by human-readable names, using a decentralized naming system called IPNS.

IPFS is useful on earth right now

Archivists

It's not enough to organize the world's information—we need to store it in a way the world can remember it. IPFS provides deduplication, high performance, and clustered persistence.

Service providers

If your company delivers large amounts of data to users, a peer-to-peer approach could save you millions in bandwidth. IPFS can provide secure peer-to-peer content delivery.

Researchers with data

If you're working with, distributing, and analyzing huge datasets, IPFS offers fast performance and decentralized archiving.

Developing world

High latency networks are a high barrier of entry to developing world. IPFS networks can provide resilient access to data and persistence offline.

Content creators

IPFS brings the freedom and independent spirit of the web at full force—and at low cost. For users in freedom-hostile states or dealing with sensitive data, IPFS is a good fit.
